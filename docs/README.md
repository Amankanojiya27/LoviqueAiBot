# Lovique Documentation

This folder is the main documentation hub for Lovique.

Use these docs when you want to understand the product, explain it to someone else, or make technical changes without losing the bigger picture.

## Start here

- [Product overview](./product-overview.md)
  Product goals, target users, value proposition, feature set, and how the experience is meant to feel.
- [Technical architecture](./technical-architecture.md)
  Frontend and backend responsibilities, request flows, data model summary, deployment notes, and operational behavior.
- [Documentation playbook](./documentation-playbook.md)
  How to keep docs updated as the product changes.

## Suggested reading order

1. Read the [product overview](./product-overview.md) if you are new to Lovique.
2. Read the [technical architecture](./technical-architecture.md) if you will change code or infrastructure.
3. Read the [documentation playbook](./documentation-playbook.md) if you will maintain or expand the docs.

## What these docs are for

These docs are meant to answer questions like:

- What is Lovique as a product?
- Who is it for?
- What are the core user flows?
- How do frontend and backend work together?
- What parts of the system are sensitive or easy to break?
- Which docs should be updated when a feature changes?

## Scope

This folder focuses on product understanding and system understanding.

It intentionally does not duplicate every setup command from the frontend and server READMEs. Those remain the source of truth for local installation and environment configuration.
